Project MindGraph：AI 情绪舒缓伴侣
阶段一：MVP（最小可行性产品）产品与技术白皮书
第一部分：产品需求文档 (PRD)
1. 产品概述
产品定位： 一款主打“极简、隐私、强陪伴”的轻量级 AI 情绪舒缓应用。内核由专业的心理学流派（如 CBT）驱动，外在表现为极具亲和力的角色扮演伴侣。

核心理念： 降低心理咨询门槛，提供随时在线的情绪托底；不做严肃医疗诊断，专注解决日常焦虑、内耗与轻度抑郁情绪。

隐私承诺（信任基石）： 实行“阅后即焚”机制。仅保留用户确认后的脱敏摘要用于长线陪伴，原始聊天记录在单次会话结束后物理销毁。

2. 核心功能模块与交互机制
2.1 痛点导向的角色选择 (Scenario-Driven Persona Selection)
放弃前置心理测试，采用直击痛点的卡片 UI，降低理解门槛。

卡片设计规范： 包含【角色人设】+【主打痛点标签】+【白话简介】。

预设 MVP 角色：

逻辑派（CBT 内核）： “邻家哥哥/学长”。标签：💬脑子一团乱麻、💬总觉得要搞砸。主打拆解问题、认知重构。

共情派（人本主义内核）： “知心学姐/树洞”。标签：💬今天就是想哭、💬不想听大道理。主打无条件接纳、情绪抱持。

深挖派（精神分析内核）： “心理学大叔”。标签：💬总是遇到同一类人、💬痛感好熟悉。主打潜意识探索、童年溯源。

2.2 隐私握手与契约建立 (Privacy Handshake)

交互： 用户首次选择角色进入对话前，弹出不可跳过的【隐私契约】。

文案核心： “这里的对话对所有人保密。会话结束后系统仅生成核心摘要，原始记录即刻销毁。摘要内容需经您确认才会保存，用于我们下次更好的相遇。”

2.3 核心对话与动态风控 (The Core Session)

自然开场： 读取上一次的摘要。如有“家庭作业（行动小尝试）”，系统不生硬查岗，而是结合当前天气、日期等上下文自然寒暄引入（例如：“今天周五啦，上次说的去便利店转转的小计划，这两天有碰到合适的机会吗？”）。

红线风控（危机拦截）： 对话全过程中，后台静默运行危机词汇/意图检测。一旦用户表达自杀、自残、伤人意图，立即挂起当前角色设定，强制输出标准危机干预话术及热线电话。

2.4 阅后即焚与摘要确认 (Post-Session Wrap-up)

摘要生成： 对话结束（用户主动点击结束或长时间未回复），系统生成结构化摘要，包括：核心情绪状态、探讨焦点、认知盲点发现、约定的微小行动（Homework）。

用户审核： 摘要向用户展示，提供三种操作：

确认并归档（保存至成长日记）。

编辑/划线删除（删除不想留存的敏感细节后再保存）。

全盘销毁（不留存任何记录）。

数据清洗： 用户操作完成后，数据库立即清空本次 Session 的原始对话记录。

2.5 极简反馈体系 (Session Rating)

交互： 摘要确认后，弹出三档评价：“这次聊天感觉怎么样？”

😊 好多了（正反馈，增加该流派匹配权重）

😐 差不多

😔 依然难受（触发特殊关怀话术）
