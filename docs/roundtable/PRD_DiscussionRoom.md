> **关联文档：** 本文档是讨论室子系统的详细设计，与 `PRD_Roundtable.md`（圆桌会谈整体项目文档）配合使用。

## 1. 概述

### 1.1 产品背景
讨论室是圆桌会谈的核心功能模块，提供多角色实时对话交互能力。用户可以选择不同身份参与讨论，与AI角色进行深度对话。

### 1.2 设计目标
- 支持用户以三种身份（主持人/参与者/旁观者）参与讨论
- 实现清晰的发言权机制和@交互逻辑
- AI主持人智能引导讨论节奏

### 1.3 核心角色定义

| 角色 | 身份说明 | 发言权限 | 交互方式 |
|------|---------|---------|---------|
| **主持人** | 功能性角色，控制讨论流程 | 完全发言权 | 直接输入，可@任意角色发言 |
| **参与者** | 讨论参与者 | 有限发言权 | 可直接发言，可@角色，主持人协调 |
| **旁观者** | 纯观察者 | 无发言权 | 仅观看，不干预讨论 |

## 2. 讨论室交互逻辑

### 2.1 三种模式的发言规则

#### 2.1.1 主持人模式

```
规则：
- 用户完全控制讨论流程
- 输入内容直接进入对话流（显示为"主持人"）
- 可使用@角色名指令指定某个角色发言
- 角色不会主动发言，必须被@才会回复

@指令响应：
- @角色名 → 该角色立即回复
- @主持人 → 主持人（AI）回应
```

#### 2.1.2 参与者模式（重点澄清）

```
规则：
- 用户可以直接发言到讨论群，其他角色能看到
- 输入内容直接显示在对话流中（显示为"你"）
- 支持@特定角色，被@的角色会直接答复

发言触发逻辑（重要）：
┌─────────────────────────────────────────────────────────────┐
│ 参与者发言后的处理流程：                                      │
│                                                             │
│ 情况A：参与者没有@任何角色                                   │
│   → 发言直接显示在对话流                                     │
│   → AI主持人决定是否@某个角色进行回应                         │
│   → 如果主持人@了某角色，该角色回复                           │
│   → 如果主持人不@，则仅参与者发言，讨论继续                    │
│                                                             │
│ 情况B：参与者@了特定角色                                     │
│   → 发言直接显示在对话流                                     │
│   → 被@的角色直接答复参与者                                  │
│   → AI主持人可选择是否继续@其他角色                          │
│                                                             │
└─────────────────────────────────────────────────────────────┘

示例：
[你]: 我认为仁政比法治更重要
[主持人]: @孟子 孟子如何看待这位参与者的观点？
[孟子]: 仁政确为治国之本，然法治亦不可或缺...

或者（参与者@角色）：
[你]: @孔子 请问仁的核心是什么？
[孔子]: 仁者爱人，己所不欲勿施于人...
```

#### 2.1.3 旁观者模式

```
规则：
- 纯观看模式，无法发言
- AI主持人自主推进讨论
- 输入框禁用或隐藏
- 显示"[旁观者模式 - 仅供观看]"提示
```

## 3. @指令系统

### 3.1 @指令格式

```
@角色名 发言内容

示例：
@项羽 谈谈您为什么选择乌江自刎
@孔子 您觉得仁政的核心是什么
@主持人 结束这个话题吧
```

### 3.2 @指令解析规则

```python
规则：
1. 输入以@开头，识别为目标角色
2. @后面的第一个连续字符串（不含空格）为角色名
3. 角色名后的所有内容作为发言内容
4. 如果角色不存在，返回提示"未找到角色：xxx"

示例解析：
输入 "@项羽 谈谈" → {target: "项羽", content: "谈谈"}
输入 "@孔子 仁政的核心" → {target: "孔子", content: "仁政的核心"}
输入 "你好@刘邦" → {target: None, content: "你好@刘邦"}  # 不以@开头，不触发
```

### 3.3 @指令响应矩阵

| 发言者 \ 被@者 | @AI角色 | @主持人 | 无@ |
|---------------|---------|---------|-----|
| **主持人** | AI角色回复 | AI主持人回复 | 主持人自己记录 |
| **参与者** | AI角色直接答复 | AI主持人回复 | 显示发言，主持人决定是否引导 |
| **旁观者** | 不响应 | 不响应 | 不响应 |

## 6. AI主持人行为规范

### 6.1 主持人核心职责

```
1. 开场引导：介绍讨论话题，参与角色
2. 进程控制：推进讨论节奏，引导不同角色发言
3. 内容整合：串联不同角色的观点，进行过渡
4. 规则维护：确保讨论围绕主题，控制发言顺序
```

### 6.2 主持人发言模板

```python
# 开场模板
开场 = [
    "各位，今日我们聚首讨论「{topic}」。",
    "在场的有：{characters}。",
    "让我们就此话题展开探讨。",
]

# 邀请角色发言
邀请发言 = "@{character}，您对「{topic}」有何见解？"

# 回应参与者（无@的情况）
回应参与者 = "这位参与者提到了「{point}」，{response}"

# 串联过渡
串联 = "方才{character1}所言颇有见地，{character2}您怎么看？"

# 推进讨论
推进 = "关于「{topic}」，还有哪些方面值得探讨？"
```

### 6.3 主持人发言触发时机

```
1. 讨论开始时：自动开场介绍（仅非主持人模式）
2. 参与者无@发言后：主持人可选择是否@角色回应
3. 角色发言后：决定是否@下一个角色
4. 讨论冷场时：主动提问推进
5. 讨论充分时：引导总结收尾
```

```python
        """开场介绍"""

    def should_respond_to_player(self, message: str) -> bool:
        """判断是否要回应参与者（参与者无@时）"""

    def decide_next_speaker(self, context: DiscussionContext) -> str | None:
        """决定下一个发言角色（可返回None表示不强制要求角色回复）"""

    def generate_response_to_player(self, message: str) -> str:
        """生成对参与者的回应（当选择回应时）"""

    def should_end_discussion(self) -> bool:
        """判断是否应该结束讨论"""
```

### 9.2 参与者模式核心逻辑

```python
def handle_participant_message(message: str):
    """
    参与者模式消息处理流程
    """
    # 1. 解析是否@了角色
    parsed = parse_mention(message)
    mentioned = parsed['target']

    # 2. 将参与者发言加入对话流
    add_to_discussion(speaker="player", content=message)

    # 3. 决策：是否需要AI主持人响应
    if mentioned:
        # 情况B：@了角色，被@角色直接答复
        trigger_character_response(mentioned, message)
        # 主持人可选择是否继续引导
        host_may_continue = True
    else:
        # 情况A：无@，主持人决定是否引导
        if host_agent.should_respond_to_player(message):
            response = host_agent.generate_response_to_player(message)
            add_to_discussion(speaker="host", content=response)

            # 主持人决定是否@角色
            next_speaker = host_agent.decide_next_speaker()
            if next_speaker:
                trigger_character_response(next_speaker)
        else:
            # 主持人选择不响应，直接继续
            pass
```
